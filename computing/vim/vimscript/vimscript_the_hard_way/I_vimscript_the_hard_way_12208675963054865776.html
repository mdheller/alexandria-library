    • Introduction

    • 16.1 Normal
      • :normal
      • Takes a set of characters
      • ... perform whatever operation
      • ... as if you're in normal mode
      • Don't understand special characters
        • Example
          • <cr>
          • <space>
          • ...
      • Talk about the bang (!) later
    • + - 16.2 Execute
      • :execute
      • Take a Vimscript string
      • ... performs it as a command
      • Example
        • :execute "write"
        • :execute ":normal! gg"
        • :execute ":normal! gg/a<cr>"
        • ... useful on the last one
        • ... since normal doesn't understand
        • ... "special characters"
      • Substitute every special characters
        • Example
          • <cr> to \r
      • .. so we can do
        • :normal! ?^==\+$<cr>:nohlsearch<cr>kvg_
      • ... other pieces of the command
        • ?^==\+$
          • Search backward
          • ... for any line with two "=" or more
          • ... and nothing else
        • :nohlsearch
          • Clear the previous search
        • kvg_
          • k
            • More up one line
          • v
            • Enter visual mode
          • g_
            • move to the last non blank character
            • ... of the current line
            • Use instead of $
            • ... not to highlight newline character

    • 22.1 Case Sensitivity
      • Behavior of if can depends on user settings
      • Example
        • With ignorecase
          • :set noignorecase
          • :if "foo" == "FOO"
            • false
          • :if "foo" == "foo"
            • true
        • Without ignorecase
          • :set ignorecase
          • :if "foo" == "FOO"
            • true

    • 22.2 Code Defensively
      • You can NEVER trust comparison operator
      • ... when writing plugins
        • NEVER use "=="
      • Two other comparison operators
        • ==?
          • Case insensitive
          • ... no matter the user has set
        • ==#
          • Case sensitive
          • ... no matter the user has set
      • When comparing integers, doesn't matter
      • ... still, use case insensitive feels better
        • Problem if you change your integer
        • ... to a string!
